## 现状（事实）

- **渲染方式**: 工作经历不是纯 Markdown 直出，而是在 `src/components/Preview.tsx` 中按数据结构渲染（Experience → Project → ContributionGroup → List）。
- **主要样式来源**: `src/styles/base.css`，其中 `.resume h3/h4`、`.contrib-title`、`.resume li` 的字号/粗细/间距较接近，导致层级差异弱；嵌套列表与父列表共享同一套 spacing 规则，导致密度高、难扫读。

## 信息主次：你想让读者先看到什么

建议把每段经历拆成“扫读主干 + 细节补充”两层：

- **主干（必须一眼看到）**
  - 公司（或公司+业务线）
  - 岗位
  - 时间段（右侧对齐即可）
  - 一句话职责（responsibility）
  - 每个项目 2-4 条“成果型要点”（含指标）
- **次要（可快速略过但不丢）**
  - 项目 summary（背景描述）
  - 技术栈 stack（更像 meta，不应该抢注意力）
  - 分组标题（重构/提效/质量等）——应是“组织信息”，不是“标题信息”

## 层级映射（建议默认规则）

以当前结构为基准，建立“权重梯度”：

- **L1 章节标题**: `工作经历`（`h2`）——保持现有风格即可
- **L2 经历标题**: 公司/岗位/时间（目前在 `h3.exp-title`）——需要最清晰
- **L3 项目标题**: 项目名（`h4`）——清晰但弱于 L2
- **L4 分组标题**: 例如“重构&迭代/效率提升/质量保障”（`.contrib-title`）——建议改成更轻量的“标签/前缀”视觉
- **L5 要点列表**: 成果 bullets（`li`）——可读性第一，间距不宜过大
- **L6 meta 行**: summary/stack（`p.project-desc / p.project-stack`）——更淡、更紧凑

## 版式与间距（解决“密集但没主次”的关键）

### 1) 经验块之间要“分段”

- 每段 Experience 之间至少有一个稳定的视觉断点：
  - **选项 A（ATS 最友好）**: 增加留白（margin）
  - **选项 B**: 极淡分隔线（1px，浅灰）

### 2) 标题与正文的间距要“递减”

- 越靠近内容细节（分组/子列表），间距越紧凑。
- 父列表（项目层）的 `li` 间距略大于子列表（贡献分组）`li` 间距。

### 3) 嵌套列表要“显式分层”

- 子列表缩进可略小（避免一层层向右挤压造成拥挤）。
- 子列表 `li` 的 margin-bottom 应小于父列表。

## 内容写作约束（从源头降密度）

建议默认写作规范（不强制，但作为推荐）：

- **每个项目**:
  - 1 句 summary（可选）
  - 1 行 stack（可选）
  - **2-4 条**成果型要点（尽量量化）
  - 分组标题 **2-3 个以内**，避免“分组太多反而像目录”

## 3 套可选视觉风格（你选一个方向）

### 风格 1：标准（默认推荐）

- L2（经历标题）明显；L3（项目）清晰；L4（分组）弱化成“行内标签”
- summary/stack 降权，成为更淡的 meta
- 适合大多数信息量

### 风格 2：紧凑（适合内容多、页数卡得紧）

- 缩小 Experience/Project 的垂直间距
- 子列表更紧凑
- 仍保持层级差异，但“呼吸感”更少

### 风格 3：强调成果（适合你这类量化很多的简历）

- 对 bullets 中的数字/百分比/单位做轻度强调（例如半粗体或更深色）
- 其他文案更淡，强化“成果扫读”

## 分阶段落地路线（后续执行）

### Phase 1（仅 CSS，最低风险，最快见效）

- 目标：不改 DOM，仅通过 `base.css` 建立层级与分层列表 spacing
- 预期收益：整体“像目录/像报纸”的层级感明显提升

验收口径：

- 一眼能区分：经历标题 > 项目标题 > 分组 > 要点
- 技术栈/背景描述不会和成果要点抢注意力
- A4 预览/打印/PDF 不出现明显溢出或分页异常

### Phase 2（微调 Preview 结构，提升扫读路径）

- 目标：把经历头信息改成更符合扫读的布局（公司/时间优先，岗位次级），并把项目 meta 行做更紧凑的呈现
- 说明：仍保持单栏、文本可复制、结构语义清晰

### 可选增强

- **密度开关**: 标准/紧凑切换（通过 CSS 变量/类名）
- **指标高亮**: 仅对量化片段强调，不制造“全是粗体”的噪音
